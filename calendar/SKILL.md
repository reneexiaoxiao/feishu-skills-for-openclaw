# 飞书日程管理

## 操作类型识别

| 用户意图 | 操作类型 | 必填参数 |
|---------|---------|---------|
| 创建日程 | create | summary, start_time, end_time |
| 查询日程 | list | start_time, end_time |
| 更新日程 | update | event_id, 更新字段 |
| 删除日程 | delete | event_id |

## 工具调用

### 创建日程（create）

**用户输入**：
```
创建明天下午2点的日程，主题"项目会议"
```

**API 调用**：
```json
{
  "action": "create",
  "summary": "项目会议",
  "start_time": {
    "timestamp": "1674206400"
  },
  "end_time": {
    "timestamp": "1674210000"
  }
}
```

### 查询日程（list）

**用户输入**：
```
查看这周的日程安排
```

**API 调用**：
```json
{
  "action": "list",
  "start_time": "2026-03-11 00:00",
  "end_time": "2026-03-17 23:59"
}
```

### 更新日程（update）

**用户输入**：
```
将会议时间改为下午3点
```

**API 调用**：
```json
{
  "action": "update",
  "event_id": "evt_xxx",
  "start_time": {
    "timestamp": "1674210000"
  },
  "end_time": {
    "timestamp": "1674213600"
  }
}
```

### 删除日程（delete）

**用户输入**：
```
删除明天的会议
```

**API 调用**：
```json
{
  "action": "delete",
  "event_id": "evt_xxx"
}
```

## 执行流程

### 1. 识别操作类型

根据关键词判断：

| 关键词 | 操作 |
|--------|------|
| 创建、新建、添加、安排 | create |
| 查看、查询、获取、有哪些 | list |
| 更新、修改、改、调整 | update |
| 删除、取消、移除 | delete |

### 2. 提取时间信息

**绝对时间**：
```
"2026-03-12 14:00"
"3月12日下午2点"
```

**相对时间**：
```
"明天下午2点" → 转换为具体日期时间
"下周一上午" → 转换为具体日期时间
"后天" → 转换为具体日期
```

### 3. 提取其他信息

- **标题/主题**：summary
- **参会人**：attendees（需要转换为 user_id）
- **地点**：location
- **描述**：description
- **时长**：计算 end_time

### 4. 构建并执行API调用

## 时间处理

### 相对时间转换

| 用户输入 | 转换结果 |
|---------|---------|
| 今天 | 当前日期 |
| 明天 | 当前日期 + 1天 |
| 后天 | 当前日期 + 2天 |
| 下周一 | 下一个周一 |
| 本周五 | 本周周五 |
| 下个月 | 下个月同日 |

### 时间格式

**时间点**：
```
下午2点 → 14:00
上午9点 → 09:00
中午12点 → 12:00
```

**时长**：
```
1小时 → start + 1小时
半小时 → start + 30分钟
2小时 → start + 2小时
```

## 参会人处理

### 从人名获取 user_id

**用户提供人名**：
```
参会人：张三、李四
```

**处理**：
1. 搜索"张三" → ou_xxx
2. 搜索"李四" → ou_yyy
3. 构建 attendees 数组

### 直接使用 user_id

```
参会人：ou_xxx, ou_yyy
```

## 错误处理

### 错误1：时间格式错误

**处理**：
```
"时间格式不正确
正确格式：
- "2026-03-12 14:00"
- "明天下午2点"
- "下周一上午"
```

### 错误2：开始时间晚于结束时间

**处理**：
```
"开始时间不能晚于结束时间
请检查时间设置"
```

### 错误3：参会人不存在

**处理**：
```
"找不到参会人：XXX
请检查：
1. 用户名是否正确
2. 或直接提供 user_id"
```

### 错误4：日程冲突

**处理**：
```
"该时间段已有日程：
- 14:00-15:00 项目会议

是否继续创建？"
```

### 错误5：无权限操作

**处理**：
```
"无权限操作此日程
可能原因：
1. 不是日程创建者
2. 参会人无编辑权限"
```

## 注意事项

1. **时间格式**：支持相对时间和绝对时间
2. **时长计算**：默认时长1小时
3. **参会人**：需要 user_id（ou_开头）
4. **权限要求**：创建/编辑需要相应权限
5. **日程冲突**：建议提醒用户检查冲突
