# 发送飞书文本消息 - 路由技能

## 🎯 技能定位

这是官方插件的**路由技能**，帮助虾识别用户意图并调用正确的工具。

---

## 🚨 智能提示逻辑

### 场景1：用户提供姓名（不是ID）

**用户输入**："给张三发消息'你好'"

**虾的响应**：
```
我可以帮你发送消息！为了准确发送，请：
1. 在对话中 @张三（这样我能获取到准确的 open_id）
2. 或者提供张三的 open_id（格式：ou_xxx）
```

**原因**：直接发送需要 open_id，但 @ 可以让系统自动获取

---

### 场景2：用户提供群名（不是ID）

**用户输入**："给产品群发消息'开会通知'"

**虾的响应**：
```
我可以帮你发送消息！为了准确发送，请：
1. 在对话中 @产品群
2. 或者提供群的 chat_id（格式：oc_xxx）
```

---

### 场景3：用户提供ID（可以直接发送）

**用户输入**："给 ou_1234567890 发消息'你好'"

**虾的响应**：
直接调用 feishu_im_user_message 发送消息 ✅

---

### 场景4：在群聊中@某人（最佳实践）

**用户输入**：在群里 @张三 并说"告诉他开会了"

**虾的响应**：
```
收到！我会：
1. 从 @ 标签中获取张三的 open_id
2. 发送消息："开会了"
```

---

## 📋 调用示例

### 示例1：直接发送（有ID）

```json
{
  "action": "send",
  "receive_id": "ou_1234567890",
  "receive_id_type": "open_id",
  "msg_type": "text",
  "content": "{\"text\":\"你好\"}"
}
```

### 示例2：发送到群聊

```json
{
  "action": "send",
  "receive_id": "oc_1234567890",
  "receive_id_type": "chat_id",
  "msg_type": "text",
  "content": "{\"text\":\"大家好\"}"
}
```

---

## 🔑 ID 类型说明

| 类型 | 前缀 | receive_id_type | 获取方式 |
|------|------|-----------------|---------|
| 群聊 | oc_ | chat_id | @群名 或 直接提供 |
| 用户 | ou_ | open_id | @用户名 或 直接提供 |

**推荐**：使用 @ 方式，让系统自动解析 ID ✅

---

## 💡 最佳实践

**对用户说**：
- ✅ "你可以 @张三 来发送消息"
- ✅ "请在对话中 @产品群"
- ✅ "我需要群聊的 chat_id，你可以 @群来获取"

**不要说**：
- ❌ "请提供 receive_id"
- ❌ "需要 open_id 参数"
- ❌ "缺少必要参数"

---

## 🎯 关键价值

1. **降低门槛** - 用户不需要知道 ID 格式，用 @ 即可
2. **友好提示** - 引导用户使用最简单的方式
3. **路由作用** - 帮助虾找到正确的官方工具
